專案模板

快速套用 starter 模板，直接生成角色、世界觀、風格與章節範例，適合開新專案或測試流程。

套用後請重新索引，新的資產才會進入本地知識庫。

小說工作流

借鏡 novelWriter / Manuskript 的場景式寫作方式，先整理角色、世界觀、關係與事件，再回頭審查章節。

1. 先建設定
角色卡、世界觀、關係圖、時間軸與伏筆都維持 Markdown / JSON，本地可版本化。
2. 再寫場景
每次審查都以單一章節或場景為單位，現在也能直接從 data/chapters/ 載入與儲存章節。
3. 最後回補結構
審查結果可直接回寫到時間軸、伏筆追蹤與角色關係，形成可追蹤的修稿紀錄。

本地 RAG 模式

建議使用順序

  1. 在 data/characters/ 新增角色設定 .md 檔案
  2. 在 data/worldbuilding/ 新增世界觀設定
  3. 點擊左側「重新索引」，建立本地知識庫
  4. 前往「一致性審查」從章節檔案庫載入章節，或直接貼上單一場景內容
  5. 審查完成後，直接把重點回寫到「關係圖 / 時間軸 / 伏筆追蹤」，讓下一次審查有更穩定的上下文
